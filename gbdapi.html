How it works

You send it a region, a venue type, and the menu items you're deciding between. It scores each one from 0–100 on how likely it is to show up with rucola, radicchio, or a "guarnito" of something leafy you didn't ask for. Higher is riskier.

1

Pull the dropdown options

Call ?action=get_config once to get the valid list of regions and venue types. Don't hardcode these — build your form from the response.

2

Collect the menu items

Each item needs a desc (the menu description, ideally in Italian since the keyword list is Italian) and a price.

3

Send everything to assess

Region, venue, and the items array (JSON-stringified) go in the query string. You get back every item annotated with its full score breakdown.

·

get_config

?action=get_config

Returns the current list of supported regions and venue types, so a dropdown never falls out of sync with the scoring model.

assess

?action=assess®ion=ROMA&venue=RISTORANTE&items=[...]

Scores a list of dishes against a region and venue context.

region — one of the values from get_config, required.
venue — one of the values from get_config, required.
items — JSON array, URL-encoded, e.g. [{"desc":"tagliata con rucola","price":24}]
Response (one item){ "desc": "tagliata con rucola", "price": 24, "risk": 68.5, "regionBonus": 0.30, "venueBonus": 0.30, "priceBonus": 0.225, "priceDiff": 9, "priceSensitivity": 0.025, "wordCount": 3, "wordBonus": 0.045, "keywordTotal": 1.20, "matchedKeywords": [ { "word": "rucola", "mod": 1.00 }, { "word": "con", "mod": 0.20 } ], "reg": "ROMA", "ven": "RISTORANTE" }

The final risk is every bonus summed, then clamped to 0–100. A dish can rack up more raw score than that internally — the clamp just means "100% risky" is the ceiling, not a promise the math stopped there.

Scoring model

Five inputs sum into one score: region, venue, price relative to a €15 baseline, description length, and keyword hits.
