🔋 Universal Battery Cloud Engine API

Transform raw battery measurements into actionable insights

Overview

The Universal Battery Cloud Engine is a RESTful API that converts raw battery measurements into meaningful percentage metrics using advanced chemistry-specific models. It supports multiple battery types with intelligent input handling.

Base URL: https://script.google.com/macros/s/AKfycbxexo633Vur2zr-frU2Xo2RYThrrziD2aXL7rgvNi69I-ct3LbjnjwFrcMVE9gqezw0uA/exec

All requests are made via HTTP GET with query parameters. Responses are returned in JSON format.

Supported Battery Types

The API automatically recognizes various naming conventions for each battery type:

Lithium-Ion / LiPo

Accepts: lithium ion, li-ion, liion, nmc, lithium polymer, lipo

LiFePO₄ (LFP)

Accepts: lithium iron phosphate, lfp, lifepo4

Lithium Titanate (LTO)

Accepts: lithium titanate, lto, titanate

Sodium-Ion

Accepts: sodium ion, sodium, na-ion, na

Nickel-Metal Hydride

Accepts: nimh, nickel metal hydride

Nickel-Cadmium

Accepts: nicd, nickel cadmium

Lead-Acid

Accepts: lead, lead acid, pb

API Endpoints

1. Calculate State of Charge (SoC)

Determines the remaining charge percentage based on voltage measurements and battery chemistry.

Parameters

Parameter Type Required Description
method String Yes Set to percentage
chemistry String Yes Battery type (see supported types above)
voltage Number Yes Total pack voltage in volts
cells Integer No Number of cells in series (default: 1)
Example Request:
?method=percentage&chemistry=lithium%20polymer&voltage=12.0&cells=3

Response Format

{ "status": "success", "method": "percentage", "resolved_chemistry": "Lithium-Ion / LiPo", "total_cells": 3, "input_total_voltage": 12.0, "calculated_voltage_per_cell": 4.0, "state_of_charge_pct": 90.0 }

2. Calculate State of Health (SoH)

Estimates battery degradation based on internal resistance measurements.

Parameters

Parameter Type Required Description
method String Yes Set to health
chemistry String Yes Battery type
ir Number Yes Internal resistance in milliohms (mΩ)
Example Request:
?method=health&chemistry=sodium%20ion&ir=350

Response Format

{ "status": "success", "method": "health", "chemistry": "Sodium-Ion", "input_resistance_mohm": 350, "fresh_baseline_mohm": 45, "state_of_health_pct": 80.0 }

3. List Supported Types

Returns all supported battery types and their accepted naming conventions.

Parameters

Parameter Type Required Description
method String Yes Set to chemistries
Example Request:
?method=chemistries

Response Format

{ "status": "success", "method": "chemistries", "supported_mappings": { "nimh": "Nickel-Metal Hydride", "nicd": "Nickel-Cadmium", "li-ion": "Lithium-Ion / LiPo", "lfp": "LiFePO4", "lto": "Lithium Titanate", "na": "Sodium-Ion", "lead": "Lead-Acid (2V Cell)" }, "accepted_search_aliases": { "lithium ion": "li-ion", "li-ion": "li-ion", "nmc": "li-ion", ... } }

4. API Help

Returns API documentation and usage information.

Parameters

Parameter Type Required Description
method String No Set to help (default if no method specified)
Example Request:
?method=help or simply access the base URL

Error Handling

The API returns error responses with a status field set to "error" and a descriptive message.

{ "status": "error", "message": "Missing or unrecognized chemistry, or missing voltage parameter." }

Common Error Scenarios

  • Missing parameters: Required parameters not provided
  • Invalid chemistry: Chemistry string not recognized (check supported types)
  • Invalid values: Non-numeric values for voltage, cells, or ir

Best Practices

  1. Use consistent units: Voltage in volts (V), internal resistance in milliohms (mΩ)
  2. Specify cell count: Always provide the cells parameter for multi-cell packs to ensure accurate calculations
  3. Cache chemistry list: Call the chemistries endpoint once and cache the results rather than repeatedly querying
  4. Handle errors gracefully: Always check the status field in responses
  5. Flexible input: Use any of the accepted aliases for battery types—the API will resolve them automatically

Important Notes

Platform Considerations:
  • This API runs on Google Apps Script infrastructure
  • Execution time limits apply (typically 30 seconds per request)
  • Consider implementing client-side caching for frequently requested data
  • For high-volume applications, consider deploying to a dedicated server
